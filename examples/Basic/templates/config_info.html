To get started, the first thing you need to do is include the JST.js file present in the bin directory just before you close the body tag. Next you need to create a "config" for your JST engine to read. This config tells JST where all of the templates and providers are present, so that it can retrieve and display them. Below is the JST config block for this page.
<script type="jst/config">
	{
		"templates": {
			"heading": "$BASEURL/heading.html",
			"info": "$BASEURL/templates/info.html",
			"provider_info": "$BASEURL/templates/provider_info.html",
			"config_info": "$BASEURL/templates/config_info.html"	
		},
		"providers": {
			"info": "$BASEURL/providers/info.json"
		}
	}
</script>

As you can see, it's nothing but a script tag of type jst/config. The content the config should be a json having two parts, templates and providers. JST internally converts the $BASEURL variable to the current base url. The config can either be written inside the script tag, as mentioned above, or it can be provided an external source file. For eg, 

<script type="jst/config" src="path/to/source.json"></script>


After providing the config, each DOM element can be given with it's template and provider by using the data-template="template_name" and data-provider="provider_name" attribute.
